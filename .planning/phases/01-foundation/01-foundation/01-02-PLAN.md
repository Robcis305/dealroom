    Create src/app/api/auth/send/route.ts (POST):
    1. Parse { email } from request body, validate with zod (email string)
    2. Rate limit: authSendLimiter.limit(email) — return 429 if exceeded
    3. generateToken() → hashToken() → compute expiresAt = now + 10min
    4. DELETE existing unused tokens for this email (so user can request a fresh link)
    5. INSERT magicLinkTokens { email, tokenHash, expiresAt }
    6. Build magicLink URL: `${NEXT_PUBLIC_APP_URL}/auth/verify?token=${rawToken}&email=${email}`
    7. Send via Resend using the magic-link.tsx React Email template
    8. Return 200 { success: true }

    Create src/app/api/auth/verify/route.ts (GET):
    1. Extract token and email from searchParams
    2. Rate limit: authVerifyLimiter.limit(clientIP from x-forwarded-for header)
    3. hashToken(rawToken) → query magicLinkTokens WHERE tokenHash = hash
    4. If no row: redirect to /auth/verify?error=used (already consumed = deleted on first use)
    5. If row exists but expiresAt < now: DELETE the expired row, redirect to /auth/verify?error=expired
    6. If valid: DELETE the row (single-use), upsert user (INSERT ... ON CONFLICT DO UPDATE SET updatedAt), createSession(userId), setSessionCookie, redirect to /deals
    7. Use timingSafeTokenCompare only if doing in-memory comparison — DB lookup by hash is already safe

    Create src/app/api/workspaces/route.ts: GET calls getWorkspacesForUser(), POST calls createWorkspace() with zod-validated body { name, clientName, cisAdvisorySide, status }.
